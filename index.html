DCT — How it works

The image is divided into N×N blocks. Each block undergoes a 2D DCT — transforming pixel values into frequency coefficients. Low-frequency (smooth) info sits top-left; high-frequency (detail) sits bottom-right. Zeroing out small coefficients and running inverse DCT gives the compressed result. This is the core of JPEG compression.

Reading the visuals

Coefficient Map — bright pixels = retained data, black = discarded. Loss Map — red intensity shows where information was destroyed. Bright red clusters = blocking artifacts (DCT) or edge smearing (Wavelet). High PSNR (>40 dB) = visually lossless. Below 30 dB = visible degradation.
